# Lattice-Planner 软件著作权登记说明性文档

本目录及项目根目录下的相关文件，用于以 **Lattice-Planner** 名称申请中国软件著作权登记时的说明性材料（不含源程序）。

---

## 本目录文件一览

| 文件 | 用途 |
|------|------|
| **软件著作权申请材料清单.md** | 登记所需材料总览、源程序与文档鉴别材料要求、申请表填写要点及提交前自检清单。 |
| **软件简要说明.md** | 申请表中「软件名称、版本号、开发完成日期、首次发表日期、软件用途、技术特点、运行环境」等的填写参考；请将【】内占位改为实际日期或数值后使用。 |
| **软件设计说明书.md** | 完整的设计说明书，推荐作为「文档鉴别材料」使用；提交时取该文档的前 30 页与后 30 页（共 60 页），每页不少于 30 行。 |

---

## 项目根目录相关文件

| 文件 | 用途 |
|------|------|
| **user-manual.md** | 《Lattice-Planner 用户手册》，可作为文档鉴别材料的另一种选择（与设计说明书二选一）；亦可单独随软件提供给用户。 |
| **ARCHITECTURE.md** | 架构说明，已与《软件设计说明书》内容衔接；供开发与登记参考。 |

---

## 使用建议

1. 先阅读 **软件著作权申请材料清单.md**，了解登记流程与材料要求。  
2. 填写申请表时，对照 **软件简要说明.md** 填写软件全称、版本、完成日、用途、技术特点、运行环境等。  
3. 将 **软件设计说明书.md**（或 **user-manual.md**）转为 Word/PDF 并排版后，截取前 30 页 + 后 30 页作为文档鉴别材料提交。  
4. 源程序鉴别材料请从项目源代码中按「前 30 页 + 后 30 页」、每页不少于 50 行的要求自行整理（仅提交本软件原创代码）。

登记要求可能随政策调整，提交前请以中国版权保护中心官网最新规定为准。
